# 2026-07-19 PayNowBiz Development Log

## Work Summary

Supabase 접속 정보를 소스 코드에 직접 기록하던 방식을 다른 앱과 동일한 Vite 환경변수 및 GitHub Repository Variables 방식으로 통일했다. 운영 설정과 로컬 설정의 관리 위치를 분리하고, 키 교체 시 소스 수정 없이 재배포할 수 있도록 배포 구조를 정리했다.

## Work Completed

* 기존 Supabase URL과 Publishable Key를 GitHub Repository Variables의 `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`로 등록했다.
* `supabase.config.js`가 실제 값 대신 Vite 환경변수를 읽도록 변경했다.
* 정적 파일 직접 배포 방식을 Vite 프로덕션 빌드 산출물 배포 방식으로 전환했다.
* Supabase 클라이언트를 CDN import에서 고정 버전 npm 의존성으로 전환하고 잠금 파일을 추가했다.
* 로컬 개발용 `.env.local`은 Git 추적에서 제외하고, 설정 예시는 `.env.example`로 제공했다.
* 커스텀 도메인이 빌드 산출물에 유지되도록 `CNAME`을 Vite의 `public` 디렉터리로 이동했다.
* GitHub Variables 누락 시 배포를 중단하는 검증 단계를 추가했다.
* GitHub Variables를 주입한 프로덕션 빌드, 산출물 내 설정 주입, `CNAME` 포함 여부를 확인했다.
* Publishable Key로 Supabase Auth 설정 API를 호출해 HTTP 200 응답을 확인했고, npm 보안 감사에서 취약점 0건을 확인했다.

## Decisions

* Pay Me Mom과 기억해야 할 규칙을 통일하기 위해 환경변수 이름을 동일하게 유지했다. `VITE_SUPABASE_ANON_KEY`라는 이름을 사용하지만 실제 값은 브라우저 공개가 허용된 최신 Publishable Key다.
* Vite 5는 지원 종료 및 개발 서버 취약점이 확인되어, 현재 보안 패치가 제공되는 최소 지원 계열인 Vite 6.4.3을 고정했다.
* 운영 값은 GitHub Repository Variables에서 관리하고 로컬 값은 Git에서 제외된 `.env.local`에서 관리하도록 역할을 분리했다.
* 브라우저에 포함되는 Publishable Key는 비밀 값이 아니므로 GitHub Secrets가 아닌 Variables를 사용했다. 데이터 접근 보안은 기존과 동일하게 Supabase RLS 정책이 담당한다.

## Current Status

GitHub Variables 등록과 코드 전환이 완료됐으며, 동일한 값으로 프로덕션 빌드와 Supabase 연결 검증을 통과했다. 변경 사항은 아직 커밋하거나 배포하지 않은 상태다.
