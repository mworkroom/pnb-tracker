# 2026-07-08 PayNowBiz Tracker Development Log

## Work Summary

페이나우비즈 앱 실행 시 워크스페이스 권한 확인 단계에서 멈춘 뒤 앱을 다시 열어야 정상 진입하는 문제를 조사했다. 가격표 앱과 비교한 결과, 페이나우비즈에 최근 추가된 짧은 권한 확인 타임아웃과 자동 재시도 흐름이 느린 Supabase 응답 상황에서 요청을 겹치게 만들고, 로그인은 유지됐지만 앱 화면으로 넘어가지 못하는 상태를 만들 수 있다고 판단했다.

## Work Completed

* 워크스페이스 권한 확인을 15초 단위로 3회 재시도하던 흐름을 제거했다.
* 권한 확인은 기존처럼 단일 요청을 20초까지 기다리도록 되돌렸다.
* 권한 확인 실패 시 노출되던 수동 `다시 확인` 버튼과 관련 이벤트 처리를 제거했다.
* 가격표 앱의 관리자 인증 흐름은 단순한 세션 확인 후 권한 확인을 적용하는 구조이고, 가격표의 일반 사용자 세션은 Supabase Auth가 아니라 30일짜리 자체 JWT 세션이라는 점을 확인했다.

## Decisions

* 브라질 Supabase 프로젝트를 사용하는 환경에서는 0.5초 안팎의 초기 지연보다, 첫 실행에서 인증 상태가 반쯤 남는 문제가 더 큰 사용성 손상이라고 보았다.
* 타임아웃을 더 짧게 만들고 재시도하는 방식은 체감 속도를 개선하기보다 느린 네트워크에서 중복 요청과 혼란스러운 인증 상태를 만들 수 있어 제거했다.
* 가격표 앱의 느린 진입과 세션 확인 표시는 별도 구조의 문제이므로, 이번 변경에서는 페이나우비즈의 로그인 진입 안정화에만 범위를 제한했다.

## Current Status

* 페이나우비즈의 권한 확인 흐름은 단일 요청 기반으로 정리됐다.
* `node --check app.js`로 JavaScript 문법 검사를 통과했다.
* 실제 Supabase 응답 시간과 모바일 PWA 백그라운드 복귀 동작은 배포 환경에서 확인이 필요하다.

## Next Steps

* 배포 후 첫 실행, Google 로그인 직후 복귀, 앱을 백그라운드에 두었다가 재개하는 흐름을 실제 기기에서 확인한다.
* 가격표 앱에서 일반 사용자 세션이 몇 분 만에 풀리는 것처럼 보이면, `expiresAt` 값과 Edge Function의 `BUSINESS_SESSION_SECRET`/`CENTER_SESSION_SECRET` 변경 여부를 별도로 확인한다.
