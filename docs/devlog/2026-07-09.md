# 2026-07-09 PayNowBiz Tracker Development Log

## Work Summary

엄마가 주로 사용하는 선결제 기록 화면에서 섹션 구분이 약하고 전체 인상이 칙칙하게 보이는 문제를 개선했다. 기존 1페이지 구조는 유지하되, 회사 컬러인 블루를 상단과 입력 작업 영역에 제한적으로 적용하고 1년 경과 기록은 관리자 화면으로 이동했다.

## Work Completed

* 메인 화면 제목을 `선결제 사용내역`으로 정리하고, 사용 중인 선결제 섹션 제목을 화면에 보이도록 변경했다.
* `새 사용내역 입력` 영역을 연한 블루 배경의 작업 패널로 분리해 카드 목록과 입력 폼의 경계를 더 명확하게 만들었다.
* 메인 하단에는 승인번호 검색과 최근 완료 내역만 유지하고, 이전 기록 섹션은 제거했다.
* 관리자 화면에 `이전 기록` 탭을 추가해 1년 경과 기록을 관리자만 확인할 수 있도록 옮겼다.
* 관리자 이전 기록 탭에서 연도별 기록 로딩 상태, 연도 펼침, 카드 상세 열기 흐름을 기존 archive 데이터를 재사용하도록 연결했다.
* 이전 기록 상세에서도 불러온 archive 거래 내역이 표시되도록 거래 조회 범위를 확장했다.
* 색상 팔레트를 기존 회색 중심에서 현대적인 중립 회색과 회사 블루 accent 조합으로 조정했다.

* 회사 블루는 엄마가 바탕화면에서 앱을 찾고 회사 업무용 앱으로 인식하는 데 도움이 되므로 완전히 배제하지 않고, 상단 바와 primary action 중심으로 제한해 사용했다.
* 이전 기록은 일상 입력 흐름과 직접 관련이 낮고 관리자만 확인해도 충분하므로 메인 하단에서 제외했다.
* 입력 폼은 카드 목록과 같은 흰 카드로 두면 구분이 약해지기 때문에, 연한 블루 작업 패널로 분리해 시각적 역할을 명확히 했다.

## Work Summary

어제 로그인 안정화 수정 후에도 앱 실행 시 `워크스페이스 권한을 확인하는 중입니다` 단계에서 멈추는 문제가 반복되어 인증 흐름을 다시 조사했다. 가격표 앱은 일반 사용자 화면에서 Supabase Auth와 `workspace_members` 조회를 거치지 않고 자체 접근코드 세션으로 상품을 불러오므로, 같은 Supabase 프로젝트를 사용해도 페이나우비즈와 초기 진입 경로가 다르다는 점을 확인했다.

## Work Completed

* 페이나우비즈의 워크스페이스 권한 조회에서 `workspaces` 조인을 제거하고 `workspace_members`의 필요한 컬럼만 읽도록 단순화했다.
* 가격표 관리자용 워크스페이스인 `00000000-0000-0000-0000-000000000002`는 페이나우비즈 멤버십 후보에서 제외했다.
* 권한 확인에 한 번 성공한 사용자와 워크스페이스 정보를 `localStorage`에 캐시하도록 추가했다.
* 다음 실행부터는 저장된 권한이 있으면 앱 화면을 먼저 열고, Supabase 권한 확인은 뒤에서 다시 시도하도록 변경했다.
* 캐시된 권한으로 앱을 연 뒤 실시간 권한 확인이 지연되거나 실패해도 로그인 화면으로 되돌리지 않고 앱 사용 상태를 유지하도록 했다.

## Decisions

* 가격표 앱과 페이나우비즈의 차이는 서버 위치보다 초기 인증 방식의 차이가 더 크다고 판단했다. 가격표 일반 화면은 Edge Function 접근코드 세션만 확인하지만, 페이나우비즈는 Supabase Auth 세션 복원 후 `workspace_members` 권한 조회까지 완료되어야 앱 화면을 보여준다.
* `workspaces` 조인은 앱 진입 권한을 판단하는 데 필수 정보가 아니므로 제거했다. 조인 대상 테이블의 RLS나 네트워크 지연이 초기 진입을 막지 않게 하기 위한 결정이다.
* 느린 네트워크에서 매번 권한 확인 완료를 기다리게 하는 대신, 이미 성공한 권한은 로컬 캐시로 신뢰하고 앱 화면을 먼저 보여주는 방식이 실제 사용자 경험에 더 적합하다고 판단했다.

## Current Status

* 페이나우비즈는 저장된 멤버십 캐시가 있으면 권한 확인 화면에 머무르지 않고 앱 화면을 먼저 표시한다.
* Supabase의 최신 권한 확인이 성공하면 캐시를 갱신하고, 멤버십이 없으면 캐시를 지운 뒤 등록되지 않은 계정 안내를 보여준다.
* `node --check app.js`로 JavaScript 문법 검사를 통과했다.
* 실제 모바일 PWA에서 첫 실행, 재실행, 백그라운드 복귀 흐름은 배포 후 다시 확인이 필요하다.

## Next Steps

* 배포 후 J님의 실제 기기에서 앱을 완전히 종료했다가 다시 열 때 권한 확인 화면에 갇히지 않는지 확인한다.
* 캐시가 없는 최초 로그인 직후에도 지연이 남는다면 Supabase SQL 함수로 `get_paynowbiz_membership` 같은 전용 RPC를 만들어 RLS/조인 영향을 더 줄이는 방안을 검토한다.
