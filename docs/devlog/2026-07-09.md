# 2026-07-09 PayNowBiz Tracker Development Log

## Work Summary

엄마가 주로 사용하는 선결제 기록 화면에서 섹션 구분이 약하고 전체 인상이 칙칙하게 보이는 문제를 개선했다. 기존 1페이지 구조는 유지하되, 회사 컬러인 블루를 상단과 입력 작업 영역에 제한적으로 적용하고 1년 경과 기록은 관리자 화면으로 이동했다.

## Work Completed

* 메인 화면 제목을 `선결제 사용내역`으로 정리하고, 사용 중인 선결제 섹션 제목을 화면에 보이도록 변경했다.
* `새 사용내역 입력` 영역을 연한 블루 배경의 작업 패널로 분리해 카드 목록과 입력 폼의 경계를 더 명확하게 만들었다.
* 메인 하단에는 승인번호 검색과 최근 완료 내역만 유지하고, 이전 기록 섹션은 제거했다.
* 관리자 화면에 `이전 기록` 탭을 추가해 1년 경과 기록을 관리자만 확인할 수 있도록 옮겼다.
* 관리자 이전 기록 탭에서 연도별 기록 로딩 상태, 연도 펼침, 카드 상세 열기 흐름을 기존 archive 데이터를 재사용하도록 연결했다.
* 이전 기록 상세에서도 불러온 archive 거래 내역이 표시되도록 거래 조회 범위를 확장했다.
* 색상 팔레트를 기존 회색 중심에서 현대적인 중립 회색과 회사 블루 accent 조합으로 조정했다.
* 카드 그림자와 굵은 글자 사용을 줄이고, 카드별 포인트 컬러는 얇은 왼쪽 선과 배지 중심으로 유지했다.
* `node --check app.js`로 JavaScript 문법 검사를 통과했다.

## Decisions

* 메인 화면을 여러 페이지로 분리하면 실제 사용자인 엄마에게 새 혼란이 생길 수 있어 1페이지 구조를 유지했다.
* 회사 블루는 엄마가 바탕화면에서 앱을 찾고 회사 업무용 앱으로 인식하는 데 도움이 되므로 완전히 배제하지 않고, 상단 바와 primary action 중심으로 제한해 사용했다.
* 이전 기록은 일상 입력 흐름과 직접 관련이 낮고 관리자만 확인해도 충분하므로 메인 하단에서 제외했다.
* 입력 폼은 카드 목록과 같은 흰 카드로 두면 구분이 약해지기 때문에, 연한 블루 작업 패널로 분리해 시각적 역할을 명확히 했다.

## Current Status

* 메인 화면에는 사용 중인 선결제, 새 사용내역 입력, 승인번호 검색, 최근 완료 내역만 남아 있다.
* 이전 기록은 관리자 모드의 별도 탭에서 확인할 수 있다.
* JavaScript 문법 검사는 통과했다.
* 실제 Supabase 데이터와 로그인 상태에서의 화면 확인은 아직 별도로 수행하지 않았다.

## Next Steps

* 실제 모바일 기기에서 엄마가 카드 목록과 입력 영역을 더 쉽게 구분하는지 확인한다.
* 관리자 이전 기록 탭에서 오래된 기록의 상세 거래가 기대대로 표시되는지 배포 환경에서 확인한다.
* 필요하면 버튼 크기와 상단 블루 농도를 실제 사용 피드백에 맞춰 한 번 더 조정한다.

## Work Summary

Atomy 업무용 앱이라는 맥락을 맞추기 위해 사이트와 같은 Noto Sans 계열 폰트를 적용했다. iOS 기본 시스템 폰트가 본문에서는 무난하지만 제목과 상단 영역에서는 앱이 허술해 보일 수 있다는 판단에 따라 폰트만 별도로 조정했다.

## Work Completed

* `index.html`에 `Noto Sans`와 `Noto Sans KR` Google Fonts 로드를 추가했다.
* `styles.css`의 기본 font stack을 `Noto Sans KR`, `Noto Sans`, system font 순서로 변경했다.
* 제목, 버튼, 입력 폼, 카드 텍스트가 같은 업무용 쇼핑몰 계열 폰트 인상을 갖도록 정리했다.

## Decisions

* 두 앱 모두 Atomy 사이트를 사용하는 사람들을 위한 내부 업무 도구이므로, 브랜드 사이트와 비슷한 Noto Sans 계열을 사용하는 것이 가장 자연스럽다고 판단했다.
* 별도 장식용 제목 폰트는 엄마 사용자에게 낯설 수 있어 적용하지 않았다.

## Current Status

* 폰트 로드와 기본 font-family 설정이 반영되었다.
* JavaScript 동작에는 변경이 없다.

## Next Steps

* 실제 모바일 화면에서 제목과 카드 금액의 가독성이 더 좋아졌는지 확인한다.
